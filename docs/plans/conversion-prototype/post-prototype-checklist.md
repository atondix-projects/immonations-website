# Post-prototype production checklist

Do not treat visual approval as production clearance.

## Evidence and legal gate

- [ ] Verify all business metrics and case-study outcomes against dated records.
- [ ] Secure testimonial, review, image, video, partner, award, and property-media permissions.
- [ ] Confirm Dr. Klein wording and relationship.
- [ ] Replace every prototype-only, sample, or provisional claim that cannot be proven.
- [ ] Complete legal review of privacy, imprint, cookies, forms, disclosures, and tracking.

## Backend decisions

- [ ] Choose form/valuation data destination and retention policy.
- [ ] Define CRM fields, consent evidence, notifications, ownership, and response SLA.
- [ ] Choose listing feed/provider and document the sample-to-live migration.
- [ ] Define email delivery, abuse prevention, error handling, and monitoring.
- [ ] Decide whether Calendly remains direct or is integrated into a controlled flow.
- [ ] Implement real consent management before any non-essential storage or tracking.

## Production functionality

- [ ] Replace local valuation completion with secure submission and failure recovery.
- [ ] Replace sample listings with verified live inventory and accurate availability states.
- [ ] Implement accessible working filters, detail routes, inquiries, and empty states as approved.
- [ ] Connect contact and download forms only after privacy/consent approval.
- [ ] Add analytics with a documented measurement plan and consent behavior.
- [ ] Add monitoring, logging, uptime checks, backups, and ownership.

## Deferred product choices

- [ ] Decide whether to build the VON POLL-inspired Immonation assistant.
- [ ] Decide whether data supports an Immowelt-inspired interactive heatmap.
- [ ] Decide whether accounts, favorites, saved searches, or buyer profiles are valuable.
- [ ] Decide whether community content earns a homepage position.
- [ ] Confirm location-detail strategy and data standards before generating routes.

## Release readiness

- [ ] Complete production accessibility audit with assistive-technology checks.
- [ ] Complete performance/Core Web Vitals pass and image/video optimization.
- [ ] Validate metadata, canonical URLs, hreflang, sitemap, robots, JSON-LD, and `llms.txt` files.
- [ ] Run security and privacy review of all new endpoints and third parties.
- [ ] Run cross-browser/device regression against production-like data.
- [ ] Create content ownership, update cadence, and claim-expiry process.
- [ ] Obtain explicit production launch approval.
