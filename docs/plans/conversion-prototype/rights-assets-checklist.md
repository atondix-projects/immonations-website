# Rights and assets checklist

## Asset selection

- [x] Every used image maps back to the normalized source inventory.
- [x] The highest-quality supplied derivative is selected.
- [x] Hero and reference images have intentional mobile crops.
- [x] Logos use approved current files, safe area, and contrast.
- [x] No duplicate, obsolete, OS-metadata, or unexplained asset is shipped.
- [x] Each asset has a useful filename and alt-text decision.

## Prototype annotations

- [x] Customer photos, quotes, screenshots, and videos carry a rights-status note in the content record.
- [x] Reference property media carries a source and intended route.
- [x] Review screenshots are tracked separately from text testimonials.
- [x] Partner and award logos are flagged as provisional until cleared.
- [x] Fonts have web-use rights and required formats.
- [x] Synthetic/generated media is labelled internally and never presented as documentary evidence.

## Public-release gate

- [ ] Written usage permission exists for identifiable people.
- [ ] Property imagery has marketing/web publication permission.
- [ ] Street numbers, plates, personal papers, faces, and other sensitive details are removed or approved.
- [ ] Testimonial consent covers name, quote, image/video, channels, and duration.
- [ ] Google/review-platform screenshot usage follows platform terms.
- [ ] Partner and award trademark/logo usage is approved.
- [ ] Photographer/videographer attribution requirements are satisfied.
- [ ] All expired or uncertain assets are replaced before deployment.
